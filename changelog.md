# Changelog

All notable changes to **Innalytix** will be documented in this file.

The format is based on **Keep a Changelog** and the project follows **Semantic Versioning (SemVer)**.

# Version Numbering

Innalytix uses Semantic Versioning.

```text
MAJOR.MINOR.PATCH
```

Examples:

* **0.1.0** – Initial platform foundation
* **0.2.0** – New platform functionality
* **0.2.1** – Bug fixes
* **1.0.0** – First production release

# Release Policy

Every released version of Innalytix must include:

* Updated documentation.
* Updated database migrations (where applicable).
* Automated tests.
* Release notes.
* Version tag in Git.
* Successful build.

A release is not considered complete until all of the above requirements have been satisfied.
